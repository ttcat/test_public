概述 Overview

Doublethink Lab 鼓勵同仁使用生成式 AI（GAI）工具來提升工作效率與品質。但 GAI 也帶來實際的風險——包括準確性、資訊安全、合作對象的安全，以及組織信譽。本政策規範所有同仁在工作上使用 GAI 時必須遵守的規則，並會隨工具、風險與最佳實務持續更新。

適用範圍 Scope

適用於 Doublethink Lab 所有層級人員，包括正職、兼職、約聘、實習生及志工。凡工作相關的 GAI 使用行為，無論使用組織配發設備或個人設備，皆適用本政策。

1 · 公平性與偏見

1.1 使用 GAI 處理任務前，應評估模型偏見是否可能導致不公平的結果。若有此風險，應採取具體措施降低偏見，否則不應以 GAI 處理該任務。

為什麼重要

GAI 模型以人類產生的資料訓練，會繼承資料中的偏見——包括性別、族裔、國籍、年齡等受保護特徵的偏見。例如用 GAI 篩選或評分求職者履歷，可能產生系統性不公平的結果；用 GAI 評估資料來源或人物的可信度，也可能嵌入偏見，影響研究的完整性。

2 · 個人責任

2.1 使用者對自己所採用、發布或轉發的 GAI 產出內容負有個人責任，包括準確性、事實性、語氣與適切性。

為什麼重要

不要把判斷外包給 GAI。若你用 GAI 草擬寄給外部夥伴的郵件而內容有事實錯誤，責任在你；若你用 GAI 協助撰寫報告，模型產生幻覺事實、捏造引註或虛構引言，是你要負責抓出來。使用任何 GAI 產出前，務必閱讀、查證、修改。請把 GAI 輸出視為一位不可靠的初階助理所寫的初稿，而非成品。

3 · 資訊安全與隱私

3.1 嚴禁將任何敏感或機密資訊輸入 GAI 模型。

典型敏感資訊包括：與我們合作對抗中共惡意影響力、可能遭受跨國鎮壓的合作夥伴之姓名或個人資訊；以及敏感的組織內部資料（如人資事件）。請把輸入模型的任何內容，都視為中共眼中的黃金情報。若不確定某資訊是否屬於敏感資料，請詢問主管。

4 · 透明揭露

4.1 對內應清楚、具體地揭露 GAI 的使用情況。

  • 送審報告時明確告知研究主管 GAI 是如何被使用的——例如「以 GAI 草擬第三節，再由本人查證與編輯」，或「以 GAI 翻譯原始中文資料」。
  • 不得在未揭露情況下，將 GAI 產出的內容充作個人原創。涵蓋文字、程式碼、分析、摘要、翻譯與圖像。
  • 揭露有助於審稿者調整審閱標準——例如更仔細檢查幻覺內容、捏造引註或文風不一致。

4.2 對外發布的內容應依研究主管或傳播團隊的指引，決定是否及如何向外部揭露 GAI 的使用。

5 · 核准廠商

重點不在模型是否為開源，而是你使用該服務時，資料會傳到哪裡。雲端服務會把提示詞送到廠商伺服器，廠商的司法管轄區與資料處理實務決定風險高低。

5.1 禁用

任何工作用途皆不得使用總部設於中國、從中國營運或與中國有重大關聯的雲端 GAI 服務（包括資料經中國司法管轄區——例如香港——傳輸的服務）。送入這類服務的資料受中國司法管轄，可能依法被要求交予中國國家機關。

例子：DeepSeek、Kimi / Moonshot AI、百度文心一言、阿里 Qwen 雲端服務（通義千問）、字節跳動豆包、騰訊混元、智譜 ChatGLM 雲端服務。

將開源權重模型完全在 DTL 掌控的硬體上執行、且無任何網路連線流向中國關聯服務，本條並不自動禁止，但處理工作資料前須事先經政策負責人核准。

5.2 核准

核准名單上的廠商皆持有公認的國際資安與合規認證（如 ISO/IEC 27001、SOC 2 Type II、HIPAA、FedRAMP）：

  • Anthropic — Claude Enterprise / Team / API
  • OpenAI — ChatGPT Team / Enterprise / API
  • Google — Gemini（Workspace 方案）/ Vertex AI
  • Microsoft — Copilot for M365 / Azure OpenAI

其他廠商請先提報主管與政策負責人審查後再使用。

6 · 智慧財產與內容

6.1 不得將第三方受版權保護的完整內容輸入公開的 GAI 服務，除非該內容已為公開可自由使用的素材。

6.2 對於 GAI 生成的圖像、影片、音樂，發布前需確認未侵犯第三方權利。對外使用時須揭露「AI 生成」。

7 · 違規處理

違反本政策可能導致：追回本計劃的獎金、內部紀律處分，或——若涉及重大資安事件——循正式人資流程處理。若發現自己或他人無意間違規，請盡速通報政策負責人，早一步處理永遠比掩蓋風險小。

本政策為動態文件，將隨工具、風險與最佳實務演變持續更新。如有疑問，聯絡政策負責人或 TTCAT。
